# 《藻井集》产品需求文档 PRD

## 1. 产品定位

《藻井集》是一个面向学生、设计学习者、设计师和文化爱好者的敦煌藻井交互图鉴网站。

它以藻井的身体化观看、朝代图鉴、单件作品解构、纹样分析、文化取色和视觉基因卡生成为核心功能，帮助用户将藻井中的色彩、纹样、结构和文化寓意转化为可用于设计创作的视觉资源。

---

## 2. 产品目标

### 2.1 体验目标

1. 让用户通过“抬头看见藻井”的方式进入网站，而不是普通点击进入。
2. 让用户感受到藻井作为建筑顶部装饰的空间属性。
3. 让藻井图像从静态观看变成可交互、可拆解、可提取的视觉资源。

---

### 2.2 内容目标

1. 展示藻井的历史流变和朝代差异。
2. 展示代表性藻井作品。
3. 拆解藻井的结构层级、纹样构成和文化寓意。
4. 提取藻井中的主要色彩与视觉特征。
5. 生成适合设计参考的视觉基因卡。

---

### 2.3 开发目标

1. 完成一个可运行的网站原型。
2. 具备完整用户路径。
3. 页面视觉风格统一。
4. 交互逻辑清晰。
5. 数据结构可扩展。
6. 后续可继续接入摄像头识别、真实取色和 AI 生成设计建议。

---

## 3. 目标用户与需求

### 3.1 学生

**用户需求：**

1. 想快速了解藻井是什么。
2. 想理解藻井的朝代变化。
3. 想知道藻井中的纹样和结构代表什么。
4. 希望以更直观的方式学习传统文化图像。

**对应功能：**

1. 藻井文化介绍。
2. 朝代图鉴。
3. 纹样结构解剖。
4. 文化说明文本。

---

### 3.2 设计学习者与设计师

**用户需求：**

1. 想从藻井中提取设计灵感。
2. 想获得可用的色卡、纹样关键词和构图参考。
3. 想知道传统纹样如何转译为现代视觉设计资源。
4. 想把文化元素用于插画、文创、服装、视觉设计、UI、展陈等场景。

**对应功能：**

1. 藻井色卡提取。
2. 局部纹样分析。
3. 视觉基因卡生成。
4. 设计应用建议。

---

### 3.3 文化爱好者与文旅游客

**用户需求：**

1. 想以轻量方式了解敦煌藻井。
2. 想获得比普通图片浏览更有参与感的体验。
3. 想理解藻井的美学价值和文化象征。

**对应功能：**

1. 沉浸式入口。
2. 图鉴浏览。
3. 作品详情。
4. 文化寓意说明。

---

## 4. 核心用户路径

用户完整路径如下：

```text
用户进入网站
↓
黑色低照度首页
↓
用户滚动 / 鼠标上移 / 手势控制
↓
藻井图像从屏幕上方缓慢展开
↓
标题《藻井集》出现
↓
进入藻井总览页面
↓
选择朝代
↓
进入单一朝代藻井图鉴
↓
选择具体藻井作品
↓
进入作品详情页
↓
查看作品概览
↓
查看空间投射效果
↓
查看纹样结构解剖
↓
点击局部纹样或色彩区域
↓
生成对应色卡与文化说明
↓
生成藻井视觉基因卡
↓
用于学习记录或设计参考
```

## 5. 功能需求优先级

## 5.1 P0：第一版必须完成

### 功能 1：首页 / 抬头看见藻井

**目标：**
用交互方式模拟“抬头看见藻井”的身体经验。

**需求说明：**

1. 页面默认黑色或低照度背景。
2. 初始状态下藻井图像位于屏幕上方或被遮挡。
3. 用户滚动、鼠标上移或移动鼠标时，藻井图像逐渐上移或展开。
4. 页面出现引导文字，例如：

   * “向上滑动，抬头看见藻井”
   * “Move upward to see the caisson ceiling”
5. 动效结束后出现主标题《藻井集》。
6. 用户可点击进入总览页面。

**第一版实现方式：**

* 使用鼠标位置或 scroll progress 控制图像位移。
* 暂不强制接入摄像头识别。

**验收标准：**

1. 首页具有明显的暗场氛围。
2. 藻井图像能随用户操作产生上移或展开效果。
3. 用户能理解“抬头观看”的交互隐喻。
4. 可顺利进入总览页。

---

### 功能 2：藻井总览页面

**目标：**
展示《藻井集》的整体入口，让用户按朝代进入图鉴。

**需求说明：**

1. 页面展示项目标题和简短说明。
2. 展示朝代入口，可采用环形图鉴、卡片阵列或横向空间卡片。
3. 朝代入口包括：

   * 北朝
   * 隋
   * 初唐
   * 盛唐
   * 中唐
   * 晚唐
   * 五代
   * 宋
   * 西夏
   * 元
4. 用户点击朝代后进入对应朝代图鉴页。
5. 页面整体保持黑色背景、低照度、高级感和文化氛围。

**验收标准：**

1. 用户可以清楚看到不同朝代入口。
2. 点击朝代后可以进入对应图鉴页面。
3. 页面视觉与首页风格一致。
4. 页面不应显得像普通资料列表，而应具有“图鉴系统”的感觉。

---

### 功能 3：单一朝代藻井图鉴

**目标：**
展示某一朝代下的代表性藻井作品。

**需求说明：**

1. 页面顶部显示当前朝代名称，例如“初唐藻井图鉴”。
2. 展示该朝代藻井作品卡片。
3. 每个卡片至少包含：

   * 藻井名称
   * 朝代
   * 洞窟编号
   * 主图
   * 一句话视觉特征
   * 关键词
4. 卡片可以横向排列、层叠排列或空间透视排列。
5. 用户点击卡片后进入具体藻井详情页。
6. 提供返回总览页的按钮。

**验收标准：**

1. 用户能明确知道自己所在的朝代图鉴。
2. 用户能浏览该朝代的藻井卡片。
3. 点击卡片能进入作品详情页。
4. 页面具有图鉴感和空间感。

---

### 功能 4：具体藻井作品详情页

**目标：**
展示单件藻井的完整信息，并引导用户进入结构、色彩、纹样分析。

**需求说明：**

详情页应包含以下内容区：

1. **作品概览**

   * 名称
   * 朝代
   * 洞窟编号
   * 简介
   * 主图
   * 关键词

2. **空间投射**

   * 展示藻井在空间中的顶部观看关系。
   * 可通过倾斜、透视、缩放等方式模拟藻井投射到顶部空间的感觉。

3. **色彩分析**

   * 展示藻井主图或局部纹样。
   * 用户点击色彩区域或预设色点。
   * 页面展示对应色卡、HEX 色值、色彩说明和文化寓意。

4. **结构解剖**

   * 展示纹样分层图。
   * 标注中心纹样、边饰纹样、方井结构、重复纹样等。
   * 点击或 hover 标注点时显示解释。

5. **视觉基因卡**

   * 根据当前藻井作品生成最终卡片。
   * 卡片包含颜色、纹样、结构、文化寓意、设计应用建议。

**页面导航建议：**

详情页顶部或右上方包含 Tab：

* 首页 / 概览
* 投射
* 色彩
* 结构
* 生成

**验收标准：**

1. 页面内容层级清楚。
2. 用户能从概览顺利进入色彩和结构分析。
3. 用户能生成视觉基因卡。
4. 页面视觉与前序页面统一。

---

### 功能 5：藻井色卡提取展示

**目标：**
将藻井中的局部色彩转化为可理解、可复用的文化语义色卡。

**需求说明：**

1. 用户点击藻井图像中的局部纹样或色彩区域。
2. 系统显示对应的预设色卡。
3. 色卡包含：

   * 色块
   * 色名
   * HEX 色值
   * 色彩角色，例如主色、辅助色、强调色
   * 对应纹样位置
   * 文化说明
   * 设计应用建议
4. 第一版使用预设色彩数据，不做实时图像取色。

**验收标准：**

1. 点击区域后色卡内容发生变化。
2. 色卡信息完整。
3. 色彩展示清晰、美观。
4. 用户能理解这些颜色如何来自藻井，并如何被用于设计。

---

### 功能 6：纹样结构解剖展示

**目标：**
帮助用户理解藻井的结构层级和纹样含义。

**需求说明：**

1. 展示藻井的结构解剖图。
2. 标注不同区域，例如：

   * 中心莲花
   * 三兔共耳
   * 飞天队列
   * 卷草纹
   * 方井结构
   * 边饰纹样
   * 层叠结构
3. 用户 hover 或点击标注时，显示对应说明。
4. 说明内容包括：

   * 纹样名称
   * 所在位置
   * 视觉特征
   * 文化含义
   * 设计转译方向

**验收标准：**

1. 结构区域标注清晰。
2. 标注交互可用。
3. 说明文字简洁，不堆砌。
4. 用户能理解藻井不是单张图案，而是由多个层级共同构成。

---

### 功能 7：视觉基因卡生成

**目标：**
形成项目的最终输出闭环。

**需求说明：**

用户在具体藻井作品页面点击“生成视觉基因卡”后，系统生成一张卡片。

卡片内容包括：

1. 藻井名称。
2. 朝代。
3. 洞窟编号。
4. 主图缩略图。
5. 主色卡。
6. HEX 色值。
7. 纹样关键词。
8. 结构关键词。
9. 文化寓意。
10. 设计应用建议。

**建议文案：**

“生成藻井视觉基因卡”
“将色彩、纹样与文化语义转化为可用于设计创作的灵感卡。”

**验收标准：**

1. 用户可以点击生成卡片。
2. 卡片内容来源于当前藻井数据。
3. 卡片视觉完成度高，适合作为项目展示成果。
4. 卡片能体现“设计转译”而不只是普通色卡。

---

## 5.2 P1：第二阶段可做

### 功能 8：摄像头手势识别

**目标：**
用摄像头识别用户手部上移动作，进一步强化“抬头观看”的身体交互。

**建议实现：**

1. 使用 MediaPipe Hands。
2. 请求摄像头权限。
3. 识别手部关键点。
4. 根据手部上移距离控制藻井图像上移。
5. 提供关闭摄像头和改用鼠标模式的选项。

**注意：**
该功能作为增强项，不影响 MVP 主流程。

---

### 功能 9：真实图片取色

**目标：**
从藻井图像中自动提取主色。

**建议实现：**

1. 使用 ColorThief、Vibrant.js 或 Canvas API。
2. 对整张图片或局部区域提取颜色。
3. 自动生成色卡。

**注意：**
第一版不建议依赖真实取色，因为文物图片可能存在光照、压缩、截图偏色等问题，会影响视觉结果稳定性。

---

### 功能 10：AI 生成设计建议

**目标：**
根据藻井色彩、纹样和文化说明，自动生成现代设计应用建议。

**建议实现：**

1. 用户选择应用场景，例如插画、文创、服装、UI、包装。
2. 系统生成设计建议。
3. 后续可接入大模型 API。

**注意：**
第一版可用预设文案模拟，不必真实接入 API。

---

## 6. 内容需求

### 6.1 每个朝代需要的数据

每个朝代至少包含：

1. 朝代 ID。
2. 朝代名称。
3. 英文名称。
4. 简介。
5. 该时期藻井视觉特征。
6. 代表作品列表。
7. 朝代主色倾向。
8. 朝代关键词。

---

### 6.2 每个藻井作品需要的数据

每个藻井作品至少包含：

1. 作品 ID。
2. 名称。
3. 朝代。
4. 洞窟编号。
5. 简介。
6. 主图。
7. 详情图。
8. 结构解剖图。
9. 色彩区域图。
10. 关键词。
11. 纹样说明。
12. 结构说明。
13. 色卡数据。
14. 文化寓意。
15. 设计应用建议。

---

### 6.3 色卡数据要求

每组色卡至少包含：

1. 色名。
2. HEX 色值。
3. 色彩角色。
4. 对应纹样区域。
5. 色彩说明。
6. 文化说明。
7. 设计建议。

示例：

```json
{
  "name": "朱砂红",
  "hex": "#B92126",
  "role": "主色",
  "region": "边饰纹样",
  "meaning": "强化藻井中的热烈、庄重与宗教装饰性",
  "designUsage": "适合用于主视觉标题、包装强调色、节庆类文创视觉"
}
```

## 7. 视觉风格需求

### 7.1 整体风格

1. 黑色背景。
2. 低照度氛围。
3. 金色或暖色文字点缀。
4. 视觉上保持神秘、安静、沉浸。
5. 避免过度国潮、霓虹、卡通化。
6. 保留敦煌色彩本身的厚重感。

---

### 7.2 页面风格

1. 首页强调沉浸式观看。
2. 总览页强调图鉴系统。
3. 朝代页强调卡片空间感。
4. 详情页强调信息清晰和文化分析。
5. 色卡页强调设计工具感。
6. 视觉基因卡强调成果感和可分享感。

---

## 9. 非功能需求

### 9.1 可维护性

1. 所有藻井内容必须数据化。
2. 页面组件与数据分离。
3. 新增藻井作品时不应修改大量页面代码。
4. 图像路径统一管理。

---

### 9.2 响应式

第一版优先适配桌面端，因为该项目适合大屏展示。
移动端需要保证基础可浏览，但不必完全还原所有空间交互。

---

### 9.3 性能

1. 图片需压缩。
2. 首页动效不能过度卡顿。
3. 图鉴卡片数量第一版不宜过多。
4. 大图可使用懒加载。

---

## 10. 风险与控制

### 风险 1：摄像头识别开发成本高

**控制方式：**
第一版用鼠标和滚动替代，摄像头作为增强功能。

---

### 风险 2：真实取色结果不稳定

**控制方式：**
第一版使用预设色卡数据，保证视觉和说明质量。

---

### 风险 3：内容过多导致开发失控

**控制方式：**
第一版只选取少量代表性藻井，先跑通完整路径。

---

### 风险 4：文化说明可能不准确

**控制方式：**
所有具体历史、洞窟、纹样解释后续需要人工核验和标注来源。

---

## 11. 第一版验收标准

第一版完成后，需要满足：

1. 网站可以正常运行。
2. 用户可以从首页进入总览页。
3. 用户可以选择朝代。
4. 用户可以进入单件藻井作品详情。
5. 用户可以查看作品介绍、结构分析、色彩分析。
6. 用户可以生成视觉基因卡。
7. 页面视觉风格统一。
8. 交互路径完整。
9. 项目能够清晰讲出“观看—解构—取色—再造”的逻辑。
10. 项目能够体现 AI 辅助设计与文化设计创新的结合。
