# docs/03-interaction-flow.md

# 《藻井集》交互流程文档

## 1. 文档目的

本文档用于明确《藻井集》网站的整体交互路径、页面切换逻辑、用户操作方式、状态变化和 Codex 开发时需要遵守的交互规则。

本项目不是普通资料展示网站，而是一个围绕“身体化观看—图鉴浏览—单件解构—色彩提取—视觉转译”展开的交互网站。因此，交互流程需要服务于项目的核心体验：

```text
抬头看见藻井
↓
进入藻井空间
↓
按朝代浏览图鉴
↓
选择某一朝代
↓
浏览该朝代代表性藻井
↓
选择具体藻井
↓
理解作品、结构、色彩与文化寓意
↓
生成视觉基因卡
```

---

## 2. 总体交互逻辑

网站的交互方式分为三类：

## 2.1 点击切换

用于主要页面之间的明确跳转。

包括：

1. 首页进入总览。
2. 总览页面进入按朝代划分的藻井图鉴。
3. 按朝代划分的藻井图鉴进入选中朝代藻井图鉴。
4. 详情页内部 Tab 切换。
5. 色彩热点、纹样热点、生成视觉基因卡等操作。

---

## 2.2 滚动展示

用于选中朝代藻井图鉴到具体藻井介绍之间的过渡。

用户进入某一朝代的藻井图鉴后，不是立刻点击进入详情页，而是先通过鼠标滚动自然浏览该朝代的代表性藻井。

当滚动到该图鉴区域的极限位置后，页面进入“鼠标滑动点选”状态。

---

## 2.3 鼠标滑动点选

用于从选中朝代藻井图鉴中选择具体藻井作品。

当用户滚动到选中朝代图鉴的末端后：

1. 鼠标水平移动会控制当前高亮的藻井卡片。
2. 当前卡片放大、前移或亮度提高。
3. 用户点击当前卡片。
4. 页面进入具体藻井介绍页。

该交互用于强化“图鉴浏览”到“作品选中”的过程，而不是简单列表点击。

---

## 3. 页面与状态关系

第一版建议采用“路由 + 页面状态”的混合方式。

```text
/
首页入口

/atlas
藻井总览页面
包含 overview 与 dynasty-index 两种状态

/atlas/:dynastyId
选中朝代藻井图鉴页面

/work/:workId
具体藻井作品详情页面
包含 overview / projection / color / structure / gene-card 等内部状态
```

## 5. 分阶段交互说明

# Phase 0：首页入口

## 5.1 页面名称

首页入口 / 沉浸式加载页

## 5.2 对应页面

```text
/
```

## 5.3 交互目标

让用户通过模拟“抬头”的动作进入网站，建立藻井作为顶部空间装饰的身体观看经验。

## 5.4 初始状态

1. 页面为黑色或低照度背景。
2. 藻井图像位于屏幕上方、较暗或部分隐藏。
3. 页面中央或底部出现轻提示：

   * “向上滑动，抬头看见藻井”
   * “Move upward to see the caisson ceiling.”

## 5.5 用户操作

第一版支持：

1. 鼠标滚轮向上 / 向下滚动。
2. 鼠标向上移动。
3. 触控板滑动。

第二阶段可支持：

1. 摄像头识别手部上移动作。
2. MediaPipe Hands 手势识别。

## 5.6 状态变化

```text
idle
↓
revealing
↓
revealed
↓
readyToEnter
```

说明：

* `idle`：页面刚加载，藻井未完全出现。
* `revealing`：用户正在滚动或移动鼠标，藻井开始上移展开。
* `revealed`：藻井完整显现。
* `readyToEnter`：标题和进入按钮出现，用户可以进入总览。

## 5.7 动效要求

1. 藻井图像缓慢上移或由暗到亮显现。
2. 不要突然出现。
3. 标题《藻井集》应在藻井出现后淡入。
4. 动效应克制、安静、有仪式感。
5. 不要使用过度霓虹、闪烁、爆炸式动效。

## 5.8 进入条件

用户点击：

```text
进入图鉴
```

或：

```text
进入藻井集
```

跳转到：

```text
/atlas
```

# Phase 1：藻井总览页面

## 6.1 页面名称

总览页面

## 6.2 对应页面

```text
/atlas
```

## 6.3 页面状态

```ts
atlasView = "overview"
```

## 6.4 交互目标

让用户从整体上理解《藻井集》是一个藻井图鉴与设计转译网站，并引导用户进入按朝代划分的藻井图鉴。

## 6.5 页面内容

1. 项目标题：

   ```text
   藻井集
   ```

2. 项目副标题：

   ```text
   面向设计转译的敦煌藻井视觉图鉴
   ```

3. 简短说明：

   ```text
   从藻井的空间结构、纹样秩序与文化色彩中，提取可用于当代设计创作的视觉基因。
   ```

4. 主视觉：

   * 环形图鉴。
   * 暗场背景。
   * 中央保留留白或一句英文提示。
   * 周围分布若干藻井或图像卡片。

5. 入口按钮：

   ```text
   进入藻井图鉴
   ```

## 6.6 用户操作

用户点击总览页面中的入口按钮或主视觉图鉴区域。

## 6.7 页面切换

```text
总览页面
↓ 点击
按朝代划分的藻井图鉴
```

在代码上可以有两种实现：

### 方式 A：同一路由内状态切换

```ts
setAtlasView("dynasty-index")
```

页面仍在：

```text
/atlas
```

### 方式 B：使用 hash 或 query 标记

```text
/atlas#dynasty-index
```

或：

```text
/atlas?view=dynasty-index
```

第一版推荐方式 A，开发更直接。

# Phase 3：选中朝代藻井图鉴

## 8.1 页面名称

选中朝代藻井图鉴

## 8.2 对应页面

```text
/atlas/:dynastyId
```

例如：

```text
/atlas/early-tang
```

## 8.3 页面含义

“选中朝代藻井图鉴”指：展示某一选中朝代中的所有代表性藻井作品。

例如用户选择“初唐”后，该页面展示初唐时期的代表性藻井作品，而不是只展示一个藻井。

## 8.4 交互目标

让用户通过滚动自然浏览该朝代藻井作品，并在滚动到末端后，通过鼠标滑动点选具体作品。

## 8.5 页面内容

1. 返回按钮。

2. 当前朝代名称：

   ```text
   初唐藻井图鉴
   ```

3. 当前朝代简介。

4. 该朝代视觉特征关键词。

5. 多张代表性藻井卡片。

6. 滚动提示：

   ```text
   滚动浏览该朝代的代表性藻井
   ```

7. 到达末端后的点选提示：

   ```text
   移动鼠标，选择一件藻井作品
   ```

## 8.6 初始交互：滚动自然展示

用户进入页面后，使用鼠标滚轮或触控板滚动。

页面中的藻井卡片可以采用：

1. 横向展开。
2. 层叠展开。
3. 景深排列。
4. 由远到近出现。
5. 随滚动逐渐展开卡片间距。

## 8.7 滚动行为

```text
用户滚动
↓
藻井卡片逐渐展开
↓
页面展示更多代表性藻井
↓
滚动进度达到 100%
↓
进入鼠标滑动点选状态
```

## 8.8 滚动状态

```ts
type DynastyGalleryScrollState =
  | "intro"
  | "scrolling"
  | "scrollEnd"
  | "selecting";
```

说明：

* `intro`：页面刚进入。
* `scrolling`：用户正在滚动浏览卡片。
* `scrollEnd`：滚动到当前区域极限。
* `selecting`：进入鼠标滑动点选状态。

## 8.9 鼠标滑动点选

当滚动到极限后，页面不立刻跳转，而是进入点选模式。

用户移动鼠标时：

1. 根据鼠标横向位置计算当前选中卡片。
2. 当前卡片高亮。
3. 当前卡片放大或前移。
4. 其他卡片降低亮度或后移。
5. 用户点击当前卡片后进入作品详情页。

## 8.10 点选逻辑

```text
滚动到末端
↓
页面提示“移动鼠标，选择一件藻井作品”
↓
用户水平移动鼠标
↓
根据 pointerX 映射 activeWorkIndex
↓
高亮对应卡片
↓
用户点击
↓
进入 /work/:workId
```

## 8.11 伪代码逻辑

```ts
const progress = scrollProgress; // 0 - 1

if (progress < 1) {
  galleryMode = "scrolling";
}

if (progress >= 1) {
  galleryMode = "selecting";
}

if (galleryMode === "selecting") {
  const activeIndex = Math.floor(pointerX / windowWidth * works.length);
  setActiveWorkId(works[activeIndex].id);
}

onClickActiveWork(() => {
  navigate(`/work/${activeWorkId}`);
});
```

## 8.12 交互兜底

如果鼠标滑动点选不好实现，MVP 可以保留以下兜底方式：

1. 卡片仍然可以直接点击。
2. 左右箭头切换当前卡片。
3. 键盘方向键切换。
4. 移动端改为横向滑动卡片后点击。

但桌面端第一优先级仍是：

```text
滚动自然展示
↓
滚动到极限
↓
鼠标滑动点选
```

这一阶段的交互重点不再是大幅空间切换，而是清晰地理解单件藻井：

```text

```

顶部或右上角使用 Tab / 胶囊导航：

```text
概览
投射
色彩
结构
生成
```

也可以沿用视觉稿中的导航：

```text

```

```text
概览 / 投射 / 色彩 / 结构 / 生成
```

# Phase 5：色彩提取交互

## 10.1 所在页面

```text
/work/:workId
```

内部状态：

```ts
activeTab = "color"
```

## 10.2 交互目标

用户点击藻井局部色彩区域，获得对应的文化语义色卡。

## 10.3 用户操作

1. 点击色点。
2. 点击图像局部区域。
3. 点击预设色块。

## 10.4 页面反馈

用户点击后：

1. 当前区域高亮。
2. 右侧或下方色卡更新。
3. 显示色名、HEX、色彩角色。
4. 显示文化说明。
5. 显示设计应用建议。

## 10.5 交互流程

```text
进入色彩模块
↓
展示藻井局部图和色点
↓
用户点击某个色点
↓
系统读取对应 paletteGroup
↓
更新色卡面板
↓
展示文化说明与设计建议
```

## 10.6 MVP 说明

第一版使用预设色卡数据。

不做：

1. 实时图片取色。
2. 用户框选区域取色。
3. AI 自动生成色彩说明。

这些作为第二阶段功能。

# Phase 7：视觉基因卡生成交互

## 12.1 所在页面

```text
/work/:workId
```

内部状态：

```ts
activeTab = "gene"
```

## 12.2 交互目标

将当前藻井的色彩、纹样、结构和文化信息整合为设计转译结果。

## 12.3 用户操作

点击按钮：

```text
生成藻井视觉基因卡
```

## 12.4 页面反馈

1. 按钮状态变化为“生成中”。
2. 卡片淡入。
3. 展示当前藻井的视觉基因信息。
4. 可以提供“返回色彩”“返回结构”等辅助按钮。

## 12.5 卡片内容

视觉基因卡包含：

1. 藻井名称。
2. 朝代。
3. 洞窟编号。
4. 主图缩略图。
5. 主色卡。
6. HEX 色值。
7. 纹样关键词。
8. 结构关键词。
9. 文化寓意。
10. 设计应用建议。

## 12.6 MVP 说明

第一版生成卡片本质是读取当前作品的预设数据并渲染，不需要接入 AI API。

## 14. 交互优先级

## 14.1 P0 必须完成

1. 首页藻井上移动效。
2. 首页点击进入总览。
3. 总览点击进入朝代图鉴。
4. 朝代图鉴点击进入选中朝代图鉴。
5. 选中朝代图鉴滚动展示。
6. 滚动末端后可选择具体作品。
7. 具体作品详情页 Tab 点击切换。
8. 色彩热点点击。
9. 纹样热点点击。

## 15. 交互设计原则

1. **先身体，后知识。**
   用户先通过“抬头看见藻井”进入体验，再阅读文化内容。

2. **先图鉴，后单件。**
   用户先看到朝代和作品群，再进入具体分析。

3. **先浏览，后点选。**
   选中朝代页面先通过滚动浏览，滚动到极限后再点选具体作品。

4. **先理解，后生成。**
   用户先查看结构、纹样和色彩，最后生成视觉基因卡。

5. **交互要克制。**
   项目气质是沉浸、安静、文化感，不做过度游戏化或过度炫技的动效。

## 16. 开发验收标准

1. 用户可以从首页完整走到视觉基因卡生成。
2. 总览页面到朝代图鉴必须是点击切换。
3. 朝代图鉴到选中朝代图鉴必须是点击切换。
4. 选中朝代图鉴到具体藻井介绍必须体现滚动浏览过程。
5. 滚动到极限后，用户可以通过鼠标滑动或兜底点击选择具体藻井。
6. 作品详情页内部以点击为主。
7. 色彩和结构模块有明确点击反馈。
8. 页面切换不应突兀，应有轻微淡入、位移或景深变化。
9. 所有交互都应有兜底方案，保证展示稳定。
10. 第一版不依赖摄像头识别和真实取色，也可以完整运行。

## 权威修正：加载页到朝代图鉴的真实路径

后续开发必须优先遵守 `docs/11-authoritative-flow-correction.md`。如果本文件前文与该文档冲突，以 `docs/11-authoritative-flow-correction.md` 为准。

当前权威路径为：

```text
/ 首页加载页
对应 04加载页4(3).svg
↓ 向下滚动
/atlas overview 总览页
对应 05总览页面(1).svg
↓ 点击总览中的单张藻井卡片
/atlas dynasty-index 朝代图鉴页
对应 06藻井图鉴（朝代）(1).svg
↓ 点击某一朝代
/atlas/:dynastyId 选中朝代藻井图鉴
对应 07选中朝代藻井图鉴(1).svg
↓ 滚动浏览或兜底点击具体作品
/work/:workId 具体藻井介绍
对应 08具体藻井介绍(1).svg
```

修正点：

1. 首页“进入图鉴”按钮只作为兜底，不是主路径。
2. 首页主路径是继续向下滚动进入 `/atlas`。
3. `/atlas` 的 `overview` 展示藻井作品/图像卡片，不展示朝代卡片。
4. 点击 `overview` 中的藻井卡片后，才切换到 `dynasty-index`。
5. `dynasty-index` 才展示朝代入口。
